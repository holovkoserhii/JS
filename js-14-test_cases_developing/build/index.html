1) Зовнішня форма карток відповідає закладеній версткою концепції
Завантажую сторінку - співставляю зовнішній вигляд окремої картки з DOM-деревом та правилами CSS, що повинні до нього діяти - повинно співпадати.

2) При завантаженні стартової сторінки з'являється поле для введення URL майбутньої закладки, а також кнопка Add.
Завантажую сторінку - перераховані два елемента повинні бути присутні у верхній частині таблиці.

***read***
3) При завантаженні стартової сторінки відразу з'являються картки раніше доданих сайтів із закладок. В даному випадку вони приїжджають із localStorage.
Завантажую сторінку - переглядаю кількість на наповнення карток - повинно співпадати з відповідними об'єктами із localStorage/sitesArray

***create***
4) Після введення url система перевіряє, чи валідна ця адреса. Якщо не валідна - видає alert. Якщо валідна, робить додаткові перевірки. У випадку їх успішного проходження додає нову картку у верхню частину секції result.
4.1. Вводимо ukr.net, натискаємо Add - валідна, додається картка.
4.2. Вводимо ukrnet, натискаємо Add - невалідна, бачимо alert, картка не додається.
4.3. Вводимо unian.ua, натискаємо Add - валідна, додається картка.
4.4. Вводимо unian.ua7777777, натискаємо Add - невалідна, бачимо alert, картка не додається.

5) Після введення url система перевіряє, чи є ця адреса у закладках даного користувача. Якщо вже присутня - видає alert. Її немає, робить додаткові перевірки. У випадку їх успішного проходження додає нову картку у верхню частину секції result.
Передумова: сайту https://github.com/ немає серед закладок.
5.1. Вводимо https://github.com/, натискаємо Add - валідна, додається картка.
5.2. Вводимо https://github.com/ повторно, натискаємо Add - бачимо alert, картка не додається.

6) До закладок додаються лише існуючі сайти.
У поле url вводжу http://non-existing-site.com/ (неіснуючий сайт) - нічого не відбувається, картка не створюється.

***delete***
7) При натисненні кнопки Remove конкретної картки - видаляється саме ця картка, інші картки зсуваються на її місце.
